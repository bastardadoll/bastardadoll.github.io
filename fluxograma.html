Estrutura Modular da Rede Social Reprograma-T / Orby Chat

Este documento apresenta uma estrutura modular e didática para a rede social, detalhando suas principais páginas, componentes e fluxos. O objetivo é fornecer uma visão clara de como as diferentes partes da plataforma se interconectam, servindo como um guia para entendimento e desenvolvimento, sem ser um diagrama técnico exaustivo.

Representação Visual da Estrutura Modular

Detalhes da Estrutura

I. Estrutura Geral / Layout Base

Componente: LayoutPrincipal

Função: Define a estrutura visual comum a quase todas as páginas após o login.

Módulos Internos: Header/BarraSuperior, Sidebar/MenuLateral, AreaConteudoPrincipal, SidebarDireita (Opcional).

Dados: Informações do usuário logado.

Ações: Navegação entre módulos/páginas.

II. Módulo: Autenticação e Onboarding

Página: PaginaLogin

Função: Permitir que usuários existentes acessem suas contas.

Componentes: FormularioLogin, BotoesLoginSocial, Link para Cadastro, Link para Recuperação de Senha.

Dados: Interage com a tabela usuarios.

Ações: Validar credenciais, redirecionar para Feed/Dashboard.

Página: PaginaCadastro

Função: Permitir que novos usuários criem contas.

Componentes: FormularioCadastro, BotoesLoginSocial, Link para Login.

Dados: Cria novo registro na tabela usuarios.

Ações: Validar dados, criar conta, redirecionar para Onboarding ou Feed.

Página: PaginaRecuperacaoSenha

Função: Guiar o usuário no processo de redefinição de senha.

Componentes: Formulário de Email, Formulário Nova Senha (após validação por token).

Dados: Interage com usuarios, sistema de tokens de recuperação.

Ações: Enviar email com link, validar token, atualizar senha.

Fluxo: OnboardingInicial

Função: Apresentar a plataforma, coletar interesses iniciais, sugerir conexões/grupos, explicar funcionalidades únicas.

Componentes: PassoBoasVindas, SeletorInteresses, SugestoesIniciais, TutorialGuiado.

Dados: Atualiza usuarios, busca sugestões.

Ações: Guiar o usuário, coletar preferências.

III. Módulo: Feed e Interações Sociais

Página: PaginaFeedPrincipal

Função: Exibir o fluxo principal de atualizações e publicações.

Componentes: CaixaCriarPublicacao, ListaPublicacoes (Componente: PublicacaoItem).

Dados: Busca publicações, interage com tabelas: publicacoes, usuarios, amizades, grupos_membros, curtidas, comentarios.

Ações: Criar nova publicação, curtir, comentar, compartilhar, ver detalhes.

Componente: PublicacaoItem

Função: Renderizar uma única publicação no feed.

Módulos Internos: Cabeçalho, Conteúdo (Texto, Imagem, Vídeo, etc.), Ações, Seção de Comentários.

Dados: Dados de uma publicação específica, comentários associados.

Ações: Disparar ações de curtir, comentar, compartilhar.

Página: PaginaPublicacaoDetalhada

Função: Exibir uma única publicação com todos os comentários e interações.

Componentes: PublicacaoItem (completo), ListaComentariosCompleta, FormularioComentario.

Dados: Dados da publicação, todos os comentários.

Ações: Comentar, responder a comentários.

IV. Módulo: Perfis de Usuário

Página: PaginaPerfilUsuario

Função: Exibir informações e atividades de um usuário.

Componentes: CabecalhoPerfil, NavegacaoAbasPerfil, ConteudoAbaSelecionada, TermometroEmocionalWidget (Opcional).

Dados: Dados das tabelas: usuarios, publicacoes, amizades, selos_usuario, checkin_emocional.

Ações: Varia conforme visualiza o próprio perfil ou o de outro.

V. Módulo: Amizades e Conexões

Página: PaginaAmigos

Função: Listar amigos do usuário.

Componentes: ListaAmigos (Componente: AmigoItem), Filtros/Busca.

Dados: Tabelas: amizades, usuarios.

Ações: Ver perfil do amigo, enviar mensagem, remover amizade.

Página: PaginaSolicitacoesAmizade

Função: Gerenciar solicitações de amizade recebidas e enviadas.

Componentes: ListaSolicitacoesRecebidas, ListaSolicitacoesEnviadas.

Dados: Tabelas: solicitacoes, usuarios.

Ações: Aceitar, recusar, cancelar solicitação.

Página: PaginaExplorarPessoas

Função: Descobrir outros usuários.

Componentes: GridPerfisSugeridos, Filtros de Busca Avançada.

Dados: Tabelas: usuarios, interesses, selos_usuario.

Ações: Visualizar perfil, enviar solicitação de amizade.

VI. Módulo: Mensagens (Chat)

Página/Componente: InterfaceChat

Função: Permitir comunicação privada síncrona/assíncrona.

Componentes: ListaConversas, JanelaChatAtiva (Componente: BalaoMensagem), CaixaInputMensagem.

Dados: Tabelas: mensagens (ou chat_mensagens_temp), usuarios, amizades.

Ações: Enviar/receber mensagens, visualizar status, iniciar novas conversas.

VII. Módulo: K-Play (Música)

Página: PaginaKPlayDashboard

Função: Hub central para atividades musicais.

Componentes: Acessos rápidos para BibliotecaPessoal, ModoAprendizado, DesafiosAtuais, RankingSemanal, ComunidadeMusical.

Página: PaginaBibliotecaKPlay

Função: Gerenciar músicas do usuário.

Componentes: ListaMusicasUsuario, BotaoUploadMusica.

Dados: Tabela: musicas (filtrada por usuário).

Ações: Upload, tocar, editar detalhes, remover música.

Página/Modal: PlayerKPlayInterativo

Função: Tocar música com letra e acordes sincronizados.

Componentes: VisualizadorLetraAcorde, Controles de Playback, DiagramaAcorde.

Dados: Dados da música selecionada.

Ações: Interagir com o player.

VIII. Módulo: Livro Conexão (Histórias CNV)

Página: PaginaLivroConexaoPrincipal

Função: Introduzir o conceito, permitir navegação/criação.

Componentes: ListaPaginasRecentes ou NavegacaoTematica, BotaoCriarPagina.

Página: PaginaLeituraCard

Função: Exibir o conteúdo de uma página/história.

Componentes: CardVisualA5, QuestionarioInterativoCNV.

Dados: Conteúdo da página (livro_paginas).

Ações: Ler, responder questionário, navegar.

IX. Módulo: Igreja Inclusiva

Página: PaginaIgrejaPrincipal

Função: Apresentar a comunidade, agenda e recursos.

Componentes: BannerBoasVindas, AgendaCultos, LinksRecursos, AcessoComunidade.

Página: PaginaCulto

Função: Permitir participação/visualização dos cultos.

Componentes: PlayerVideo, ChatAoVivo (se aplicável), AreaTextoLiturgia.

Dados: Stream/vídeo do culto, mensagens do chat.

Ações: Assistir, interagir no chat.

X. Módulo: Cursos/Aprendizado

Página: PaginaCatalogoCursos

Função: Listar cursos disponíveis.

Componentes: GridCursos (Componente: CardCurso), Filtros.

Dados: Tabela: cursos.

Ações: Visualizar detalhes, inscrever-se.

Página: PaginaCursoDetalhes

Função: Exibir informações, módulos e progresso de um curso.

Componentes: DescricaoCurso, ListaModulos, BarraProgresso, AreaDiscussaoCurso.

Dados: Dados do curso, modulos, aulas, inscricoes_cursos.

Ações: Iniciar/continuar curso, participar de discussões.

XI. Módulo: Biblioteca Colaborativa

Página: PaginaBibliotecaPrincipal

Função: Buscar e acessar recursos compartilhados.

Componentes: BarraBuscaBiblioteca, ListaRecursos, Filtros.

Dados: Tabela: biblioteca_recursos.

Ações: Buscar, filtrar, visualizar recurso.

XII. Módulo: Segurança e Moderação

Componente: BotaoDenunciar

Função: Permitir que usuários reportem conteúdo/comportamento inadequado.

Ações: Abre modal FormularioDenuncia, envia dados para moderação (tabela denuncias).

Componente: BotaoBloquear

Função: Impedir interação com um usuário específico.

Ações: Adiciona registro na tabela bloqueados.

Página: PaginaMapaVioleta

Função: Visualizar e contribuir com mapeamento de segurança.

Componentes: MapaInterativo, FormularioRelatoLocal.

Dados: Tabela: mapa_violeta_pontos.

Ações: Navegar no mapa, adicionar/avaliar ponto.

XIII. Módulo: Configurações

Página: PaginaConfiguracoes

Função: Centralizar todas as opções de personalização e gerenciamento da conta.

Componentes: MenuLateralConfiguracoes, FormularioConfiguracaoAtiva.

Dados: Tabela: usuarios e outras tabelas de configuração.

Ações: Salvar alterações nas configurações.

XIV. Elementos Transversais

Notificações

Componente: ListaNotificacoes. Sistema para informar sobre novas amizades, mensagens, interações, etc.

Dados: Tabela: notificacoes.

Busca Global

Componente: no Header para buscar usuários, grupos, conteúdo, etc.

Termômetro Emocional

Widget (Componente: SeletorEmocional) para check-in rápido.

Dados: Tabela: checkin_emocional.

Modo Foco Silencioso

Opção: Uma opção nas configurações ou um toggle rápido para ativar a interface de baixa estimulação.

Esta estrutura detalhada e modular visa organizar o desenvolvimento, permitir a reutilização de componentes e facilitar a manutenção e evolução da plataforma Reprograma-T / Orby Chat.
